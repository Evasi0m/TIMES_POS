# คู่มือเขียนบันทึกอัปเดต (รายการอัปเดต)

แคชเชียร์เห็นข้อความจากไฟล์ `src/data/updates.json` — **เขียนเป็นภาษาบ้านๆ** ไม่ใช่ commit message

## เมื่อไหร่ควรเพิ่ม patch

- มีฟีเจอร์ใหม่ที่แคชเชียร์ใช้งานได้
- แก้บั๊กที่กระทบการขาย / TikTok / ใบเสร็จ
- ปรับหน้าจอให้ใช้ง่ายขึ้นอย่างเห็นได้ชัด

ไม่จำเป็นต้องเพิ่มทุก commit เล็กๆ — เพิ่มเมื่อ **push สำคัญ** ที่อยากให้ร้านรู้

## วิธีเพิ่ม

1. เปิด `src/data/updates.json`
2. ใส่ patch **ใหม่ที่ต้นสุด** ของ array `patches` (index 0 = ล่าสุด)
3. Build + deploy ตามปกติ

## โครงสร้าง patch

```json
{
  "id": "2026-06-10-tiktok-ux",
  "date": "2026-06-10",
  "title": "ขาย TikTok ง่ายขึ้น",
  "tags": ["ใหม่", "TikTok"],
  "items": [
    "จับคู่สินค้า TikTok ได้ง่ายขึ้น ไม่ต้องเลื่อนหน้าจอเยอะ",
    "SKU ที่มี VDF/UDF ต่อท้าย ไม่โดนเตือนผิดว่าไม่ตรงแล้ว"
  ]
}
```

| ฟิลด์ | คำอธิบาย |
|-------|----------|
| `id` | ไม่ซ้ำ เช่น `YYYY-MM-DD-หัวข้อสั้น` |
| `date` | วันที่ deploy (ISO `YYYY-MM-DD`) |
| `title` | หัวข้อสั้น ภาษาพูด |
| `tags` | ประเภท + โมดูล (ดูด้านล่าง) |
| `items` | รายการ bullet ว่า **ได้อะไร** (2–5 ข้อ) |

## tags ที่ใช้ได้

**ประเภท (เลือก 1):**
- `ใหม่` — ฟีเจอร์ใหม่
- `แก้บั๊ก` — แก้ปัญหา
- `ปรับปรุง` — ใช้ง่ายขึ้น / เร็วขึ้น ไม่ใช่ฟีเจอร์ใหม่

**โมดูล (ถ้ามี):** `TikTok`, `ใบเสร็จ`, `สินค้า`, `รับของ`, `รายงาน`, ฯลฯ

## ตัวอย่างที่ดี vs ไม่ดี

| ไม่ดี (เทคนิค) | ดี (ภาษาบ้านๆ) |
|----------------|----------------|
| refactor TikTokOrderReviewPane | หน้าตรวจสอบ TikTok อ่านง่ายขึ้น |
| fix isTikTokSkuMismatch suffix | SKU ที่มี VDF ต่อท้าย ไม่โดนเตือนผิดแล้ว |
| add migration 062 | ติ๊กส่งแทนได้เมื่อลูกค้าเปลี่ยนรุ่น |

## หลัง deploy

แคชเชียร์จะเห็นจุดแจ้งเตือนบนปุ่ม **รายการอัปเดต** ใน Sidebar จนกว่าจะเปิดอ่านครั้งแรก
