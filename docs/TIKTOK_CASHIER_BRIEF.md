# คู่มือแคชเชียร์ — ออเดอร์ TikTok Shop (API)

## กฎสำคัญ (อ่านก่อนใช้งาน)

1. **ออเดอร์จาก TikTok Shop ห้าม key-in manual ที่ POS**
   - ระบบดึงออเดอร์จาก TikTok อัตโนมัติ (ทุก ~5 นาที)
   - ออเดอร์จะอยู่ในสถานะ **รอยืนยัน** จนกว่าแคชเชียร์จะยืนยัน

2. **ยืนยันออเดอร์ผ่าน badge "ออเดอร์ TikTok รอยืนยัน"** — 4 ขั้นตอน:
   - **เลือกสินค้า** — เลือกสินค้าในร้านที่จะตัดสต็อก (ไม่จำเป็นต้องตรงรุ่นที่สั่งทันที)
   - **ตรวจสอบ** — ตรวจแต่ละรายการ · ถ้าลูกค้าขอเปลี่ยนรุ่น ติ๊ก "ส่งจริงคนละรุ่น"
   - **กรอกเงิน** — ยอด net ที่ร้านได้รับจริง
   - **ยืนยัน** — ตัดสต็อก + เข้ารายงาน

3. **ถ้าระบบเตือนว่ามีออเดอร์ TikTok รอยืนยัน**
   - กด **ยกเลิก — ไปยืนยันใน TikTok** (อย่ากด "บันทึกต่อ manual")
   - ใช้ "บันทึกต่อ (manual)" เฉพาะเมื่อแน่ใจว่า **ไม่ใช่** ออเดอร์ TikTok Shop ที่ sync มาแล้ว

4. **ช่องทาง default ของ POS = หน้าร้าน**
   - เปลี่ยนเป็น TikTok/Shopee/Lazada เฉพาะเมื่อ key-in ขาย marketplace **ที่ไม่ได้มาจาก API**

## สิ่งที่แตกต่างจากขายหน้าร้าน

| | ขายหน้าร้าน | ออเดอร์ TikTok (ยืนยันที่ร้าน) |
|--|------------|---------------------------|
| ตัดสต็อก | ทันที checkout | ตอนกดยืนยัน |
| เข้ารายงาน | ทันที | หลังยืนยัน |
| ป้ายในประวัติขาย | — | "TikTok" |

## Go-live cutoff

ออเดอร์ TikTok ก่อน **13:00 วันที่ 7 มิ.ย. 2026** ไม่เข้าคิวยืนยัน (legacy — void อัตโนมัติ)

## ติดต่อ admin

- จับคู่ SKU ถาวร (super admin): E-Commerce → TikTok → แท็บ **จับคู่สินค้า**
- ปัญหา sync / webhook: แจ้ง admin ตรวจ TikTok Settings

## ตะกร้า TikTok ไม่ระบุรุ่นชัด (ค่าเริ่มต้น) + ส่งจริงคนละรุ่น

บางสินค้า TikTok มีตัวเลือกเดียว — ระบบ TikTok อาจแสดง **ค่าเริ่มต้น** แทนรหัสรุ่นชัดเจน

เมื่อเลือกสินค้าในร้านแล้ว ระบบจะให้เลือก **อย่างใดอย่างหนึ่ง** ที่ขั้นเลือกสินค้า:

| ทางเลือก | ใช้เมื่อ | ผล |
|----------|---------|-----|
| **ยืนยันการจับคู่** | รุ่นในร้านตรงกับสินค้า TikTok จริง | จำการเลือกไว้ · ออเดอร์ถัดไปเติมให้อัตโนมัติ |
| **ส่งจริงคนละรุ่น** | ลูกค้าเปลี่ยนสี/รุ่นในแชท ไม่ยกเลิกออเดอร์ | ตัดสต็อกตามในร้าน · **ไม่** จำการเลือกไว้ |

### ตัวอย่าง 4 กรณี

| กรณี | บน TikTok | ในร้าน | ทำอย่างไร |
|------|----------|--------|-----------|
| A — ค่าเริ่มต้น รุ่นเดียวกัน | ค่าเริ่มต้น | `W-218HD-1AVDF` (รุ่นถูก) | **ยืนยันการจับคู่** |
| B — ค่าเริ่มต้น แต่เปลี่ยนสีในแชท | ค่าเริ่มต้น | รุ่น/สีอื่นที่ลูกค้าตกลง | **ส่งจริงคนละรุ่น** |
| C — รุ่นชัดแต่คนละรุ่น | `AE-1600HX-3A` | `AE-1600HX-1A` | **ส่งจริงคนละรุ่น** |
| D — รหัสท้ายต่าง (รุ่นเดียวกัน) | `EF-539D-1A` | `EF-539D-1AVUDF` | ไม่ต้องทำอะไร — ระบบรู้ว่ารุ่นเดียวกัน |

### ขั้นตอน (แคชเชียร์)

1. **เลือกสินค้า** — เลือกสินค้าในร้านที่จะส่ง/ตัดสต็อก
2. ถ้า TikTok เป็นค่าเริ่มต้น หรือรุ่นไม่ตรง → เลือก **ยืนยันการจับคู่** หรือ **ส่งจริงคนละรุ่น**
3. กด **ถัดไป → ตรวจสอบ**
4. **กรอกเงิน** → ใส่ยอดที่ได้รับ → **ยืนยัน**

## ส่งจริงคนละรุ่น (Substitution)

ใช้เมื่อลูกค้า TikTok สั่งรุ่นหนึ่ง แต่ตกลงให้ส่งรุ่นอื่น **โดยไม่ยกเลิกออเดอร์บน TikTok** — ต้อง **ติ๊กเลือกเอง** ระบบจะไม่เดาให้อัตโนมัติ

### ขั้นตอน (แคชเชียร์)

1. เปิด **ออเดอร์ TikTok รอยืนยัน** → เข้าออเดอร์ที่เกี่ยวข้อง
2. **เลือกสินค้า** — เลือกสินค้าในร้านที่จะส่ง/ตัดสต็อก · ถ้ารุ่นไม่ตรง เลือก **ยืนยันการจับคู่** หรือ **ส่งจริงคนละรุ่น**
3. กด **ถัดไป → ตรวจสอบ** (หรือระบบพาไปอัตโนมัติถ้ารุ่นตรงทุกรายการ)
4. **ขั้นตรวจสอบ** — ตรวจอีกครั้ง · ติ๊ก **"ส่งจริงคนละรุ่น"** ได้ที่นี่ด้วยถ้ายังไม่ได้เลือก
5. ถ้ายังไม่เลือกทางแก้ → ยืนยันไม่ได้
6. **ขั้นกรอกเงิน** → ใส่ net → **ยืนยัน**
7. **จัดส่งของจริง** ตามที่ตกลง — label จาก TikTok อาจยังแสดง SKU เดิม

### ไม่ใช่กรณีส่งคนละรุ่น (อย่าติ๊กส่งจริงคนละรุ่น)

| สถานการณ์ | ทำอย่างไร |
|-----------|-----------|
| สต็อกในร้านหมด แต่ TikTok ยังมี | รับเข้า/ซิงค์สต็อก — **ไม่ใช่** ส่งคนละรุ่น |
| เลือกสินค้าผิดโดยไม่ตั้งใจ | เปลี่ยนให้ตรงรุ่นที่สั่ง — หรือติ๊กส่งจริงคนละรุ่นอย่างตั้งใจ |
| สต็อกในร้านไม่พอสำหรับรุ่นที่เลือก | เปลี่ยนสินค้าหรือยกเลิกบน TikTok |

### หลังยืนยัน (admin checklist)

| หัวข้อ | ทำอะไร |
|--------|--------|
| Mapping TikTok SKU เดิม | ตรวจ E-Commerce → TikTok → จับคู่สินค้า — ถ้า SKU บน order ถูก map ผิดให้แก้กลับ |
| TikTok stock SKU บน order | หลัง ship บน TikTok แพลตฟอร์มอาจตัด SKU บนใบ order — ตรวจ Seller Center ถ้า drift |
| เอกสาร | บันทึกใน order notes / แชทว่าส่งรุ่นใดแทน |

### ในประวัติขาย

- คอลัมน์ **สถานะ** แสดงจุดสีตามสถานะ — hover เพื่อดูชื่อ (ปกติ / ส่งคนละรุ่น / ยกเลิก / รอยืนยัน / รอใส่ราคา / แก้ไขแล้ว)
- บิลที่ส่งคนละรุ่นแสดงป้าย **ส่งคนละรุ่น** (ม่วง) — ในรายละเอียดบิลมี **สั่ง → ส่งจริง** ต่อบรรทัด
- ใบกำกับ/รายงานขายอิง **สินค้าที่ส่งจริง** (ถูกต้อง)

### อย่าทำ

- อย่า key-in manual ที่ POS สำหรับออเดอร์ TikTok API
- อย่าติ๊ก "ส่งจริงคนละรุ่น" เมื่อแค่สต็อก POS/TikTok ไม่เท่ากัน — ใช้เฉพาะเมื่อลูกค้าตกลงส่งรุ่นอื่นจริงๆ

## ลูกค้ายกเลิกบน TikTok ก่อนจัดส่ง (บิลไม่หาย)

**สรุป:** บิล **ไม่ถูกลบ** — ระบบเปลี่ยนเป็น `voided` อัตโนมัติเมื่อ TikTok ส่งสถานะ CANCELLED

### หaบิลในระบบ

| ที่ | วิธี |
|----|------|
| E-Commerce → TikTok Shop | แท็บ **ยกเลิก** หรือ **ทั้งหมด** · ค้นหา POS `#` หรือ TikTok Order ID ในแถบค้นหา |
| ประวัติขาย | ปิด **"ไม่รวมบิลที่ยกเลิก"** · ค้นหาเลขบิล · คอลัมน์ **สถานะ** แสดง **ยกเลิก TikTok** + เหตุผล void |

### Go-live คิวรอตีกลับ

- ระบบยืนยันสต็อกเอง (ปุ่ม **รอตีกลับ**) เริ่มนับตั้งแต่ **12 ก.ค. 2569 เวลา 20:26 น.** (เวลาไทย)
- ออเดอร์ void/ตีกลับ **ก่อน** เวลานี้ **ไม่** เข้าคิว — สต็อกที่เคลียร์ไปแล้วไม่ต้องยืนยันซ้ำ
- หลัง go-live: cancel/return ใหม่จะเข้าคิวตามปกติ

### สต็อก POS

- ถ้าร้าน **ยืนยันขายที่ POS แล้ว** (บิล `active`) แล้ว TikTok ยกเลิก/ตีกลับ → บิลเป็น `voided` ทันที แต่ **ไม่คืนสต็อกอัตโนมัติ**
- ไปที่ POS → ปุ่ม **「รอตีกลับ」** (ข้างปุ่ม TikTok รอยืนยัน) → เลือก **ได้รับของคืน** (บวกสต็อก) หรือ **ของหาย** (ไม่บวก)
- ยกเลิก **ก่อนส่ง** → ค่าเริ่มต้น「ได้รับของคืน」แต่ต้องกดยืนยัน
- ถ้ายัง **รอยืนยัน** (`pending`) → ไม่เคยตัดสต็อก จึงไม่มีอะไรคืน

### รับคืนสินค้า / เอกสารเพิ่มเติม

1. ออเดอร์ที่ **รอตีกลับ** (`stock_resolution = awaiting`) → ใช้ปุ่ม **รอตีกลับ** ที่ POS เท่านั้น (ไม่ใช่เมนูรับคืนเก่า)
2. หลังยืนยันใน card แล้ว → สถานะในประวัติขายแสดง **ได้ของคืนแล้ว** หรือ **ตีกลับ (ของหาย)**
3. ใบลดหนี้ TikTok → ยังออกผ่าน E-Commerce → แท็บคืนเงิน/คืนสินค้า (admin)

### กู้คืนเคสด่วน

1. หaบิลตามตารางด้านบน
2. ถ้าสถานะ **รอตีกลับ** → กดยืนยันใน card POS
3. ออเดอร์เก่าที่ระบบคืนสต็อก auto ไปแล้ว (ก่อนอัปเดตนี้) → backfill เป็น「ได้ของคืนแล้ว」อัตโนมัติ
